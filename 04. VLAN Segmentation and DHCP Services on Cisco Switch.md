# Cisco Layer 3 Switch Configuration Guide

## VLANs, Inter-VLAN Routing, and DHCP Configuration

---

## 📖 Overview

This document provides a step-by-step guide for performing basic Cisco switch configuration, including:

* Creating multiple VLANs
* Assigning names to VLANs
* Configuring IP addresses for VLAN interfaces
* Assigning switch ports to VLANs
* Enabling Inter-VLAN Routing
* Configuring DHCP services on a Cisco switch
* Verifying connectivity between VLANs

---

## 🎯 Objectives

By following this guide, you will learn how to:

* Create VLAN 50 and VLAN 60
* Assign descriptive names to VLANs
* Configure Switch Virtual Interfaces (SVIs)
* Assign access ports to specific VLANs
* Enable communication between VLANs
* Configure DHCP pools for automatic IP address assignment
* Save and verify switch configurations

---

# 1. Accessing the Cisco Switch

## Console Connection

Every Cisco switch includes a console port for direct management access.

### Physical Connection

1. Connect the RJ45 end of the console cable to the switch console port.
2. Connect the USB end of the console cable to the laptop.

### Login Procedure Using MobaXterm

1. Open **MobaXterm**.
2. Click **Session**.
3. Select **Serial**.
4. Choose the appropriate COM port.

Example:

```text
COM4
```

5. Click **OK**.
6. Press **Enter** several times until the switch prompt appears.

### Enter Privileged EXEC Mode

```bash
enable
```

# 2. Creating Multiple VLANs

Enter Global Configuration Mode:

```bash
configure terminal
```

or

```bash
conf t
```

### Create VLAN 50

```bash
Switch(config)# vlan 50
Switch(config-vlan)# exit
```

Switch(config)# vlan 50
Switch(config-vlan)# exit



### Create VLAN 60

```bash
Switch(config)# vlan 60
Switch(config-vlan)# exit
```

Exit VLAN configuration mode:

```bash
exit
```

### Result

Two VLANs have been successfully created:

* VLAN 50
* VLAN 60

# 3. Naming VLANs

Assigning names makes VLAN identification easier.

# 4. Assigning IP Addresses to VLAN Interfaces

Each VLAN requires a virtual interface (SVI) to function as a gateway.

---

## Configure VLAN 50 Interface

```bash
Switch(config)# interface vlan 50
Switch(config-if)# ip address 10.10.50.254 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit
```

### Gateway Information

| VLAN    | Gateway IP   |
| ------- | ------------ |
| VLAN 50 | 10.10.50.254 |

---

## Configure VLAN 60 Interface

```bash
Switch(config)# interface vlan 60
Switch(config-if)# ip address 10.10.60.254 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit
```

### Gateway Information

| VLAN    | Gateway IP   |
| ------- | ------------ |
| VLAN 60 | 10.10.60.254 |

---

## Verification

```bash
Switch(config)# do show ip interface brief
```

Expected output:

```text
Vlan50  10.10.50.254
Vlan60  10.10.60.254
```

# 5. Assigning a Single Switch Port to a VLAN

## Configure Interface Gi1/0/1 for VLAN 50

```bash
Switch(config)# interface gigabitEthernet 1/0/1

Switch(config-if)# switchport mode access

Switch(config-if)# switchport access vlan 50

Switch(config-if)# exit
```

Exit configuration mode:

```bash
exit
```

---

## Verification

```bash
Switch(config)# do show vlan
```

Expected result:

```text
Gi1/0/1 assigned to VLAN 50
```

---

# 6. Assigning Multiple Switch Ports to VLAN 50

To avoid configuring ports individually, use the Interface Range command.

## Configure Ports Gi1/0/2 through Gi1/0/6

```bash
Switch(config)# interface range gigabitEthernet 1/0/2 - 6

Switch(config-if-range)# switchport mode access

Switch(config-if-range)# switchport access vlan 50

Switch(config-if-range)# exit
```

Exit:

```bash
exit
```

### Result

Ports 2 through 6 are now members of VLAN 50.

---

## Verification

```bash
Switch(config)# do show vlan
```

Expected:

```text
Ports 1-6 belong to VLAN 50
```

# 8. Enabling Inter-VLAN Routing

## Purpose

Inter-VLAN Routing allows devices in different VLANs to communicate with one another.

### Enable Routing

```bash
Switch(config)# ip routing
```

### Result

Devices in VLAN 50 and VLAN 60 can communicate through the switch.

> ⚠️ Note: Inter-VLAN Routing requires Layer 3 functionality on the switch.

# 9. Configuring DHCP Services

DHCP automatically assigns IP addresses to connected devices.

---

## Create DHCP Pool for VLAN 50

```bash
Switch(config)# ip dhcp pool 50
```

### Define Network

```bash
Switch(dhcp-config)# network 10.10.50.0 /24
```

### Configure Default Gateway

```bash
Switch(dhcp-config)# default-router 10.10.50.254
```

Exit:

```bash
Switch(dhcp-config)# exit
```

### Result

DHCP pool for VLAN 50 has been created successfully.

---

## Create DHCP Pool for VLAN 60

```bash
Switch(config)# ip dhcp pool 60
```

### Define Network

```bash
Switch(dhcp-config)# network 10.10.60.0 /24
```

### Configure Default Gateway

```bash
Switch(dhcp-config)# default-router 10.10.60.254
```

Exit:

```bash
Switch(dhcp-config)# exit
```

### Result

DHCP pool for VLAN 60 has been created successfully.

---

### Optional
### Exclude Gateway Addresses from DHCP

Prevent DHCP from assigning gateway IPs.
```bash
Switch(config)# ip dhcp excluded-address 10.10.50.254

Switch(config)# ip dhcp excluded-address 10.10.60.254
```

### You can exclude more addresses if needed:
```bash
Switch(config)# ip dhcp excluded-address 10.10.50.1 10.10.50.20

Switch(config)# ip dhcp excluded-address 10.10.60.1 10.10.60.20
```

# 10. Saving the Configuration

Saving the configuration ensures settings remain after a reboot.

```bash
Switch# copy running-config startup-config
```

Press Enter to confirm.


OR
```bash
Switch# write memory
```

### Result

Configuration saved successfully.
