# Cisco Router Initial Configuration Guide | Connecting to Internet & LAN 

## 📖 Overview

This document describes the initial setup and configuration of a Cisco router connected to an ISP Internet connection. It includes the network topology, hardware connections, console access using PuTTY, and the initial Cisco IOS configuration.

---

# 🌐 Network Topology

## Internet Configuration

| Parameter       | Value                 |
| --------------- | --------------------- |
| Interface       | FastEthernet 0/0      |
| IP Address      | `202.123.123.2`       |
| Subnet Mask     | `255.255.255.0 (/24)` |
| Default Gateway | `202.123.123.1`       |

---

## Cisco Router LAN Configuration

| Parameter   | Value                 |
| ----------- | --------------------- |
| Interface   | FastEthernet 0/1      |
| IP Address  | `192.168.1.1`         |
| Subnet Mask | `255.255.255.0 (/24)` |

---

# 🖧 Network Topology Description

The topology used in this setup is as follows:

```text
                Internet
                    │
                    │
          ISP Ethernet Connection
                    │
          Cisco Router (FE0/0 - WAN)
                    │
          Cisco Router (FE0/1 - LAN)
                    │
               Internal LAN
```

### Connection Summary

* The Internet is connected to the Cisco router.
* The LAN is provided from the Cisco router.
* The LAN coming from the ISP is connected to the Cisco router.
* The laptop connects to the Cisco router using:

  * USB-to-Serial Adapter
  * Serial-to-RJ45 Console Cable

---

# 🛠 Required Equipment

The following hardware is required to configure the Cisco router:

* ✅ Cisco Router
* ✅ Console Cable (Serial to RJ45)
* ✅ USB-to-Serial Adapter
* ✅ Laptop or Computer
* ✅ Ethernet Cable
* ✅ PuTTY Software

---

# 🔌 Cisco Router Port Connections

| Port                            | Description                                               |
| ------------------------------- | --------------------------------------------------------- |
| **LAN - FastEthernet 0/1**      | Connect the cable that is connected with the LAN network. |
| **Console Port**                | Connect the RJ45 end of the console cable to this port.   |
| **Internet - FastEthernet 0/0** | Connect the ISP LAN connection here.                      |

# 🌐 Configuration No. 5 – Static Routing Configuration

## Overview

Static routing is configured so that all traffic destined for external networks (the Internet) is forwarded through the Internet Service Provider (ISP) gateway.

The default route (`0.0.0.0/0`) instructs the router to send all unknown destination traffic to the ISP gateway.

---

## Step 1 – Enter Global Configuration Mode

```bash
config terminal
```

---

## Step 2 – Configure the Default Static Route

Syntax:

```bash
ip route 0.0.0.0 0.0.0.0 [ISP Gateway IP Address]
```

According to the network topology:

| Parameter   | Value           |
| ----------- | --------------- |
| ISP Gateway | `202.123.123.1` |

Command:

```bash
ip route 0.0.0.0 0.0.0.0 202.123.123.1
```

---

## Step 3 – Exit to Privileged EXEC Mode

```bash
exit
```

---

## Step 4 – Verify the Routing Table

Display the routing table using:

```bash
show ip route
```

---

## Expected Result

The routing table should indicate that the **default route (`0.0.0.0/0`)** points to the ISP gateway **202.123.123.1**.

This means:

* All traffic destined for networks outside the router's directly connected interfaces will be forwarded to the ISP gateway.
* The default route is successfully configured.

# 🔥 Configuration No. 7 – Firewall NAT (Network Address Translation) Configuration

## Overview

The Internet Service Provider (ISP) device does not recognize the private LAN IP addresses configured on the router because no routing exists from the ISP to the LAN network.

To allow devices on the LAN to access the Internet, configure **Network Address Translation (NAT)** using **PAT (Port Address Translation)** with interface overload.

---

## Step 1 – Enter Global Configuration Mode

```bash
config t
```

---

## Step 2 – Create an Access List

Create a standard access list that identifies the internal LAN network.

Example output:

```text
Router_Area_1#config t
Enter configuration commands, one per line. End with CNTL/Z.

Router_Area_1(config)#access-li
Router_Area_1(config)#access-list ?

 <1-99>      IP standard access list
 <100-199>   IP extended access list
 <1100-1199> Extended 48-bit MAC address access list
 <1300-1999> IP standard access list (expanded range)
 <200-299>   Protocol type-code access list
 <2000-2699> IP extended access list (expanded range)
 <700-799>   48-bit MAC address access list
 compiled    Enable IP access-list compilation
 dynamic-extended  Extend the dynamic ACL absolute timer
 rate-limit  Simple rate-limit specific access list

Router_Area_1(config)#access-list 1
```

---

### Configure the Access List

Syntax:

```bash
access-list 1 permit [LAN Network Address] [Wildcard Mask]
```

According to the topology:

| Parameter     | Value         |
| ------------- | ------------- |
| LAN Network   | `192.168.1.0` |
| Wildcard Mask | `0.0.0.255`   |

Command:

```bash
access-list 1 permit 192.168.1.0 0.0.0.255
```

---

## Step 3 – Configure NAT Overload (PAT)

Configure NAT using the outside Internet interface.

Command:

```bash
ip nat inside source list 1 interface FastEthernet0/0 overload
```

---

## Step 4 – Configure the Outside Interface

Enter the Internet interface.

```bash
interface FastEthernet0/0
```

Configure it as the NAT Outside interface.

```bash
ip nat outside
```

---

## Step 5 – Configure the Inside Interface

Enter the LAN interface.

```bash
interface FastEthernet0/1
```

Configure it as the NAT Inside interface.

```bash
ip nat inside
```

Exit configuration mode.

```bash
exit
```

---

## Example Output


# 🚫 Configure DHCP Excluded Addresses

To reserve addresses that should never be assigned by DHCP, configure excluded addresses.

Syntax:

```bash
ip dhcp excluded-address [Start IP] [End IP]
```

Example:

```bash
ip dhcp excluded-address 192.168.1.1 192.168.1.10
```

Unplug and reconnect the LAN cable on the client computer.

The client will now receive an address **outside** the excluded range (`192.168.1.2` through `192.168.1.10`).

# 📌 Best Practices

* Configure **NAT** before allowing LAN clients to access the Internet.
* Always reserve important IP addresses (such as gateways, servers, printers, or network devices) using `ip dhcp excluded-address`.
* Verify DHCP leases using `show ip dhcp bind`.
* Confirm NAT functionality by testing Internet connectivity from a client.
* Save the running configuration with `copy running-config startup-config` or `write memory` after every successful configuration change to prevent data loss after a reboot.
