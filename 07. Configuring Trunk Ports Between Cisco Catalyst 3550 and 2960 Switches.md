# Cisco Switch Trunk Port Configuration Guide

## 📖 Overview

This document explains how to configure a **Trunk Port** between two Cisco switches to allow multiple VLANs to communicate across the network.

In this example:

* **Switch 1:** Cisco Catalyst 3550
* **Switch 2:** Cisco Catalyst 2960
* **Connection:** Port 48 ↔ Port 48
* **VLANs:** VLAN 20 and VLAN 40

---

# 📌 Objective

Configure a trunk link between two Cisco switches so that:

* VLAN 20 traffic can pass between both switches.
* VLAN 40 traffic can pass between both switches.
* Devices connected to the same VLAN on different switches can communicate successfully.

---

# 🖥️ Network Topology

```
                Cisco 3550
             +----------------+
             |                |
 VLAN20 PC1--Fa0/1            Fa0/48
             |                  |
             +------------------+
                    Trunk Link
             +------------------+
             |                  |
 VLAN20 PC2--Gi0/1            Gi0/48
             |                |
             +----------------+
                Cisco 2960
```

**Trunk Connection**

| Device     | Interface           |
| ---------- | ------------------- |
| Cisco 3550 | FastEthernet0/48    |
| Cisco 2960 | GigabitEthernet0/48 |

---

# 📋 Prerequisites

Before configuring the trunk:

* ✅ VLAN 20 already exists on both switches.
* ✅ VLAN 40 already exists on both switches.
* ✅ PCs are connected to their respective VLAN access ports.
* ✅ Console cable is available.
* ✅ PuTTY (or another terminal emulator) is installed.

---

# 🔌 Console Connection

Connect the console cable:

* RJ45 end → Cisco Switch Console Port
* USB end → Laptop

Open **PuTTY** and connect using the correct COM port.

---

# 🔍 Verify Existing VLANs

Display VLAN information:

```cisco
show vlan
```

Expected output:

* VLAN 20 exists
* VLAN 40 exists
* Port 48 is currently assigned to VLAN 1 (default)

---

# 🔍 Check Current Interface Configuration

On the Cisco 3550:

```cisco
show run interface fastEthernet 0/48
```

Current mode:

```text
switchport mode access
```

This confirms that the port is currently configured as an **Access Port**.

---

# 🔍 Check Existing Trunk Ports

```cisco
show interface trunk
```

If no trunk exists, the output will be blank.

---

# ⚙️ Configure Trunk Port on Cisco 3550

Enter Global Configuration Mode:

```cisco
enable
configure terminal
```

Select the interface:

```cisco
interface fastEthernet 0/48
```

Configure trunk encapsulation:

```cisco
switchport trunk encapsulation dot1q
```

Enable trunk mode:

```cisco
switchport mode trunk
```

Exit configuration mode:

```cisco
exit
```

---

# 📖 Explanation

### switchport trunk encapsulation dot1q

Configures IEEE 802.1Q encapsulation.

Available encapsulation types on Cisco 3550:

* dot1q (IEEE Standard)
* ISL (Cisco Proprietary)
* negotiate

For interoperability, **802.1Q** is recommended.

---

### switchport mode trunk

Changes the interface from an Access Port into a Trunk Port.

---

# ⚙️ Configure Trunk Port on Cisco 2960

Connect the console cable to the Cisco 2960.

Enter privileged mode:

```cisco
enable
configure terminal
```

Select the interface:

```cisco
interface gigabitEthernet 0/48
```

Configure trunk mode:

```cisco
switchport mode trunk
```

Disable DTP negotiation:

```cisco
switchport nonegotiate
```

Exit configuration:

```cisco
exit
```

---

# ⚠️ Why is `switchport trunk encapsulation` Not Available?

On Cisco 2950 and Cisco 2960 switches:

```text
switchport trunk encapsulation ?
```

returns:

```text
Unrecognized command
```

### Reason

Cisco 2960 switches only support **IEEE 802.1Q** encapsulation.

Since no other encapsulation method is available, the command is unnecessary.

# 📌 Allowed VLANs

Since no VLAN filtering has been configured, all VLANs are allowed across the trunk.

Example:

```text
Allowed VLANs:
1,20,40
```

Where:

* VLAN 1 = Native VLAN
* VLAN 20 = User VLAN
* VLAN 40 = User VLAN

---

# 🧪 Connectivity Test

After the trunk is established:

* PC connected to Cisco 3550 receives an IP address.
* PC connected to Cisco 2960 also receives an IP address through the network.

Verify connectivity using:

```text
ping 192.168.20.2
```

and

```text
ping 192.168.20.3
```

Successful replies confirm that traffic is traversing the trunk link between both switches.

# 💡 Notes

* Cisco 3550 supports multiple trunk encapsulation methods (ISL and 802.1Q).
* Cisco 2960 supports only IEEE 802.1Q encapsulation.
* Disabling DTP using `switchport nonegotiate` is recommended when connecting to another switch with a statically configured trunk.
* VLAN filtering can be configured later using `switchport trunk allowed vlan`.
