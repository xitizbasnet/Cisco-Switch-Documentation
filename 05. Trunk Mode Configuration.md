# Access Port vs. Trunk Port

| Feature         | Access Port                       | Trunk Port                             |
| --------------- | --------------------------------- | -------------------------------------- |
| VLAN Support    | Single VLAN                       | Multiple VLANs                         |
| Traffic Type    | Untagged                          | Tagged (802.1Q)                        |
| Typical Devices | PCs, Printers, IP Phones, Cameras | Switches, Routers, Firewalls           |
| VLAN Assignment | One VLAN only                     | Multiple VLANs simultaneously          |
| Purpose         | Connect end-user devices          | Connect network infrastructure devices |

### Access Port

An **Access Port** is used for devices that belong to a single VLAN.

Examples:

* Desktop computers
* Printers
* Security cameras
* Wireless access points (single VLAN deployments)

Traffic entering and leaving an access port is **untagged**.

### Trunk Port

A **Trunk Port** carries traffic for multiple VLANs over a single physical connection using **802.1Q VLAN tagging**.

Common uses include:

* Switch-to-switch links
* Switch-to-router links
* Switch-to-firewall links
* Connections to virtualization hosts

---

# Configuring a Trunk Port

In this example, interface **GigabitEthernet1/0/24** will be configured as a trunk port.

## Configuration

```bash
Switch(config)# interface gigabitEthernet 1/0/24
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 1,30,50
Switch(config-if)# exit
```

---

## Command Breakdown

### `switchport mode trunk`

```bash
Switch(config-if)# switchport mode trunk
```

Changes the interface from an access port to a trunk port, allowing it to carry traffic for multiple VLANs.

### `switchport trunk allowed vlan 1,30,50`

```bash
Switch(config-if)# switchport trunk allowed vlan 1,30,50
```

Specifies which VLANs are allowed to pass through the trunk link.

In this example:

* VLAN 1 is allowed
* VLAN 30 is allowed
* VLAN 50 is allowed

Any VLAN not listed will be blocked on this trunk.

# Verify Trunk Configuration

Use the following command to verify trunk status:

```bash
Switch# show interfaces trunk
```

Example output:

```bash
Port        Mode         Encapsulation  Status        Native vlan
Gi1/0/24    on           802.1q         trunking      1

Port        Vlans allowed on trunk
Gi1/0/24    1,30,50
```

This confirms that:

* The interface is operating as a trunk.
* 802.1Q tagging is being used.
* VLANs 1, 30, and 50 are allowed across the link.

---

# Best Practices

* Allow only the VLANs that are required.
* Avoid using VLAN 1 for user devices whenever possible.
* Document all trunk links and VLAN assignments.
* Verify trunk status after making configuration changes.
* Use descriptive interface descriptions for easier troubleshooting.

Example:

```bash
Switch(config)# interface gigabitEthernet 1/0/24
Switch(config-if)# description Uplink-to-Core-Switch
```

---

# Summary

* **Access Ports** carry traffic for a single VLAN and connect end devices.
* **Trunk Ports** carry traffic for multiple VLANs between network devices.
* Use `switchport mode trunk` to enable trunking.
* Use `switchport trunk allowed vlan` to control which VLANs can traverse the trunk.
* Verify your configuration with `show interfaces trunk`.

Understanding trunk ports is a fundamental networking skill and is essential for building scalable VLAN-based networks.
