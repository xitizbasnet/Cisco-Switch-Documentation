# 📖 Introduction

This document provides a step-by-step procedure to take a configuration backup of a Cisco switch using **PuTTY**.

The process includes:

* Installing/launching PuTTY
* Connecting to Cisco switches via SSH/Telnet
* Enabling session logging
* Capturing running configuration
* Saving backup files for recovery purposes

# 🌐 Cisco Switch Backup Procedure

---

# Step 1 — Gather Switch Details

Before starting the backup, collect the following information:

* Switch IP Address
* Username
* Password

---
 
# 🛠️ Step 2 — Configure PuTTY Logging

---

## 1. Open PuTTY

Launch the PuTTY application.

---

## 2. Enter Switch IP

In **Host Name (or IP address)** field:

```text id="putty_ip_01"
172.16.1.2
```

---

## 3. Enable Session Logging

Navigate in the left panel:

```text id="putty_log_01"
Session → Logging
```

---

## 4. Configure Logging Options

Select:

* ✔ **Printable output**

---

## 5. Set Log File Location

1. Click **Browse**
2. Navigate to **Desktop**
3. Name the file:

```text id="log_file_01"
172.16.1.2
```

4. Click **Save**

---

## 6. Confirm Log Path

Verify that the log file path is displayed correctly in PuTTY.

---

# 🔌 Step 3 — Access the Switch

---

## 1. Open Session

Click:

```text id="putty_open_01"
Open
```

---

## 2. Login Credentials

| Field    | Value      |
| -------- | ---------- |
| Login As | admin      |
| Username | admin      |
| Password | ********** |

---

## 3. Run Backup Commands

After login, execute:

```text id="cmd_01"
terminal datadump
```

Then run:

```text id="cmd_02"
sh run
```

---

## 📘 Command Purpose

| Command           | Purpose                        |
| ----------------- | ------------------------------ |
| terminal datadump | Enables full output logging    |
| sh run            | Displays running configuration |

---

# 💾 Saving and Viewing Backup File

---

## Step 1 — Wait for Output Completion

Ensure full configuration is displayed in PuTTY window.

---

## Step 2 — Close Session

Close PuTTY terminal after completion.

---

## Step 3 — Locate Backup File

Navigate to:

```text id="file_path_01"
Desktop → 172.16.1.2.log
```

---

## Step 4 — Open Backup File

1. Right-click file
2. Open with **Notepad**

---

## 📄 Result

The file will contain:

* Full running configuration
* Interface details
* VLAN configuration
* Routing settings
* Device-specific settings

---

# 📌 Conclusion

This procedure successfully captures and stores the running configuration of Cisco switches using PuTTY session logging.

The backup file can be used for:

* Configuration recovery
* Network audits
* Disaster recovery
* Change management documentation

---

# 🔒 Best Practices

* Store backup files securely
* Maintain versioned backups
* Perform regular configuration exports
* Restrict access to network credentials
* Validate backups periodically
