## 🤖 What is Network Automation?

When people hear *network automation*, they instantly think it means you need to be:

* ❌ A programmer
* ❌ A software engineer

That’s not what it’s like at all.

### ✅ Definition

Network automation is simply:

> Taking the network commands you already know and having a script run them for you.

***

## 🔁 Why Automation Matters

Automation makes sense when you're doing the same task repeatedly.

### 🔧 Example

* Adding a VLAN on one switch:
  ```
  VLAN 10
  ```
  ✅ Easy

* Adding VLAN 10 on many switches:

  * SSH into each device
  * Repeat commands

  ❌ Time-consuming  
  ❌ Error-prone

### ✅ Benefit

* Define once
* Execute consistently

# ✅ ✅ Final Production-Ready Script

> 📌 This is the **complete cleaned-up script** ready for real use.

```python
from netmiko import ConnectHandler

# 📌 Device configuration
switch = {
    'device_type': 'cisco_ios',
    'host': '192.168.99.1',
    'username': 'admin',
    'password': 'password'
}

# 🔗 Connect to switch
net_connect = ConnectHandler(**switch)
print(f"Connected to {switch['host']}")

# 🧑‍💻 Get hostname input
hostname = input("Enter the new hostname for the switch: ")

# ⚙️ Apply configuration
net_connect.send_config_set([f"hostname {hostname}"])

# 🔍 Verify change
output = net_connect.send_command("show run | include hostname")
print("\n🔍 Verification Output:")
print(output)

# 💾 Save configuration
net_connect.send_command("write memory")

# 🔌 Disconnect
net_connect.disconnect()

print("\n Configuration saved and connection closed")
```
