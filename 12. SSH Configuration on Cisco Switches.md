# 🔐 IT Networking Lab Manual: SSH Configuration on Cisco Switches

## 📘 Overview

In every previous lab, we have been physically connecting into the switch using a **console cable**. However, in real-world enterprise environments, this approach is not scalable.

> 💡 **Key Limitation:** You must be physically near the switch to configure it, which is impractical in large networks.

In enterprise environments, everything is managed remotely. This is where **SSH (Secure Shell)** becomes essential.

---

## 🔐 What is SSH?

SSH is a **secure network protocol** used to remotely access a device’s CLI over a network connection.

- 🔒 Encrypted communication (secure)
- 🌐 Remote access over IP networks
- 💻 Used for managing network devices like switches and routers

---

## ⚠️ Telnet vs SSH

| Protocol | Security | Encryption | Status |
|----------|----------|------------|--------|
| Telnet   | ❌ Insecure | ❌ Plain text | Deprecated |
| SSH      | ✅ Secure   | ✅ Encrypted | Recommended |

> ⚠️ **Warning:** Telnet transmits data in plain text and is not secure. It should not be used in production networks.

## 🧪 Lab Topology Overview

The goal of this lab is:

> 🎯 **PC1 will SSH into Switch 2**

### VLAN Design

- VLAN 10 → User VLAN (PC network)
- VLAN 99 → Management VLAN (SSH access)

### Network Design

- Switch 1 ↔ Switch 2 connected via **Trunk link**
- VLAN 10 and VLAN 99 allowed across trunk
- Switch 2 configured for SSH access

## 🔌 Physical & Logical Connections

### Devices

- Switch 1
- Switch 2
- PC1

### Cabling

- Switch 1 Port 48 ↔ Switch 2 Port 48 (Trunk link)
- Switch 1 Port 1 ↔ PC1 (Access port VLAN 10)

## ⚙️ Switch 1 Configuration

### 🖥️ Console Access

Connect using a console cable and open **PuTTY (COM port)**.

---

### 🧾 Basic Configuration

```bash
Switch> enable
Switch# configure terminal

Switch(config)# hostname Switch1
Switch(config)# enable secret epic password
````

---

### 📡 VLAN Configuration

```bash
Switch(config)# vlan 10
Switch(config-vlan)# name Users

Switch(config)# vlan 99
Switch(config-vlan)# name Management
```

---

### 🔌 Access Port Configuration (PC Connection)

```bash
Switch(config)# interface g1/0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
```

---

### 🔗 Trunk Configuration (Switch-to-Switch)

```bash
Switch(config)# interface g1/0/48
Switch(config-if)# switchport trunk encapsulation dot1q
```

> ⚠️ **Note:** Some devices may require explicit trunk encapsulation configuration before enabling trunk mode.

```bash
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,99
```

### 🌐 SVI (Switch Virtual Interfaces)

SVIs provide Layer 3 management access.

```bash
Switch(config)# interface vlan 10
Switch(config-if)# ip address 192.168.10.2 255.255.255.0

Switch(config)# interface vlan 99
Switch(config-if)# ip address 192.168.99.1 255.255.255.0
```

> 🔐 **Important:**
> The IP address on VLAN 99 will be used for SSH access.

---

### 🔗 Trunk Configuration

```bash
Switch(config)# interface g1/0/48
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,99
```

## 💻 PC Network Configuration (Windows)

### 📍 Path to Network Settings

Control Panel → Network and Internet → Network and Sharing Center → Change Adapter Settings

---

### ⚙️ IPv4 Configuration

Select:

* Ethernet Adapter → Properties → IPv4

Configure manually:

```text
IP Address:      192.168.10.3
Subnet Mask:     255.255.255.0 (/24)
Default Gateway: 192.168.10.2
```

> ⚠️ **Tip:** Disable firewall temporarily if ping fails during testing.

```bash

```

If both succeed:

> ✅ Layer 2 and Layer 3 connectivity is working correctly

## 🔐 SSH Configuration (Switch 2)

### 📌 Prerequisites

Before SSH configuration:

* Hostname must be configured
* Domain name must be set

---

### 🌐 Set Domain Name

```bash
Switch(config)# ip domain-name u.com
```

---

### 🔑 Generate RSA Keys

```bash
Switch(config)# crypto key generate rsa
```

Recommended key size:

```text
1024 bits
```

---

### 🔐 Enable SSH Version 2

```bash
Switch(config)# ip ssh version 2
```

> ⚠️ **Note:** SSHv1 is insecure and deprecated due to multiple design flaws.

---

### 👤 Create Local User Account

```bash
Switch(config)# username polo secret epic password
```

> 🔒 Password is stored in encrypted form using `secret`.

---

### 🧭 Configure VTY Lines (Remote Access)

```bash
Switch(config)# line vty 0 15
Switch(config-line)# login local
Switch(config-line)# transport input ssh
```

### 📝 Explanation

* `login local` → Uses local username/password database
* `transport input ssh` → Restricts access to SSH only (blocks Telnet)

## 📌 SSH Configuration Summary

To enable SSH on a Cisco switch, the following steps are required:

1. 🖥️ Configure hostname
2. 🌐 Configure domain name
3. 🔑 Generate RSA keys
4. 👤 Create local username/password
5. 🔐 Enable SSH version 2
6. 🧭 Configure VTY lines for SSH access

---

## 🚀 Key Takeaway

SSH eliminates the need for physical console access and enables **secure remote device management**, which is essential in enterprise-scale networks.
