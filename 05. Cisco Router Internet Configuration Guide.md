# 🌐 Cisco Router Internet Configuration (Professional Step-by-Step Guide)

## 📌 Overview

This document explains how to configure a Cisco router for home or office internet connectivity using a step-by-step approach. It includes physical setup, WAN configuration using DHCP, NAT configuration, connectivity verification, and saving the running configuration.

---

## 🏗️ Network Topology Description

Hello friends. In this video, I'm going to show you how to configure a Cisco router with your home or office internet step-by-step.

### 🔗 Topology Summary

- ISP internet router is already active and providing internet access.
- The LAN port of the ISP router will be connected to the Cisco router’s **FastEthernet 0/0** interface.
- The Cisco router will be configured as the WAN gateway device.
- In the next phase:
  - A Cisco switch will be added
  - VLANs will be configured
  - NAT and DHCP services will be implemented
  - A complete LAN-to-Internet architecture will be built

---

## 🔌 Physical Connectivity Setup

Now, let's proceed with the physical setup on the Cisco router.

- Connect the Ethernet cable from the ISP router LAN port to the Cisco router **FastEthernet 0/0** port.
- Connect the console cable to the router’s console port for CLI access.
- Power on and access the router via terminal.

---

## 💻 Accessing Cisco Router CLI

### 🧭 Enter Privileged EXEC Mode

```bash
Router> enable
Router#
````

---

### ⚙️ Enter Global Configuration Mode

```bash
Router# configure terminal
Router(config)#
```

---

## 🏷️ Configure Router Hostname

Change the default router name to improve identification.

```bash
Router(config)# hostname lab-router
lab-router(config)#
```

📌 Result:

* Router name changes from `Router` → `lab-router`

---

## 🌍 WAN Interface Configuration (FastEthernet 0/0)

### 🔧 Enter Interface Configuration Mode

```bash
lab-router(config)# interface FastEthernet0/0
lab-router(config-if)#
```

---

## 🌐 IP Address Configuration (DHCP Mode)

In this setup, the ISP assigns an IP address dynamically using DHCP.

### 📡 Configure DHCP on Interface

```bash
lab-router(config-if)# ip address dhcp
```

---

### 🚀 Enable the Interface

```bash
lab-router(config-if)# no shutdown
```

📌 Result:

* The interface becomes active
* IP address is assigned automatically from ISP DHCP server
* Example assigned IP: `192.168.1.7`

---

## 🔒 Configure NAT (Outside Interface)

Mark the WAN interface as the NAT outside interface.

```bash
lab-router(config-if)# ip nat outside
```

---

## 🔙 Exit Configuration Modes

```bash
lab-router(config-if)# exit
lab-router(config)# exit
lab-router#
```

---

## ✅ WAN Configuration Summary

At this stage, the Cisco router is successfully configured to receive internet connectivity from the ISP via DHCP.

---

## 🧪 Connectivity Testing

### 📍 Test 1: Verify ISP Gateway Connectivity

```bash
lab-router# ping 192.168.1.1
```

✔️ Expected Result: Successful replies

---

### 🌐 Test 2: Verify Internet Connectivity (Google DNS)

```bash
lab-router# ping 8.8.8.8
```

✔️ Expected Result: Successful replies

---

### 🌍 Test 3: Test DNS Resolution via Domain Ping

```bash
lab-router# ping www.bbc.com
```

✔️ Expected Result: Successful replies

---

## 💾 Save Configuration Permanently

To ensure the configuration persists after reboot:

```bash
lab-router# copy running-config startup-config
Destination filename [startup-config]?
```

Press **Enter** to confirm.
